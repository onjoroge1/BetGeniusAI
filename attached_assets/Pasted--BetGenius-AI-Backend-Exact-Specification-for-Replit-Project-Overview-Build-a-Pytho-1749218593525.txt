# 🎯 BetGenius AI Backend - Exact Specification for Replit

## **Project Overview**
Build a Python FastAPI backend that predicts football match outcomes using machine learning and explains predictions using OpenAI GPT-4. The app should be secure, data-driven, and return comprehensive JSON responses.

## **🚀 Core Functionality Flow**

### **API Endpoint: POST /predict**
**Input:**
```json
{
    "match_id": 12345,
    "include_analysis": true
}
```

**Process:**
1. **Authenticate** using API key in header
2. **Collect Data** from RapidAPI (team stats, form, H2H)
3. **Process Features** (goals per game, form points, etc.)
4. **ML Prediction** (probabilities for win/draw/loss)
5. **AI Analysis** (OpenAI explains the prediction)
6. **Return JSON** with complete prediction + analysis

**Output JSON:**
```json
{
    "match_info": {
        "home_team": "Arsenal",
```

## **📊 Machine Learning Approach**

### **Features to Extract:**
- **Team Strength:** Goals per game, goals against, league position
- **Recent Form:** Points from last 5 games, goal difference trend
- **Head-to-Head:** Historical results between teams
- **Home Advantage:** Home team win percentage
- **Context:** Rest days, injuries, venue factors

### **Models to Build:**
1. **Random Forest Classifier** for match outcome prediction
2. **Confidence Calculator** based on feature consistency
3. **Value Assessment** comparing AI odds vs bookmaker odds

### **Training Data:**
- Start with sample historical Premier League data
- Features: team stats, form, H2H, venue
- Target: match results (0=away win, 1=draw, 2=home win)

## **🔄 Testing Strategy**

### **Test Cases:**
1. **Valid Request:** Correct API key + valid match ID → Success response
2. **Invalid API Key:** Wrong key → 401 Unauthorized
3. **Invalid Match ID:** Non-existent match → 404 Not Found
4. **ML Model Test:** Known historical match → Verify prediction accuracy
5. **OpenAI Integration:** Valid prediction → Verify explanation quality

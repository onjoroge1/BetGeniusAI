## One leftover warning to fix

```
Failed to collect odds for match 1340892: aggregate function calls cannot be nested
```

That’s a Postgres rule: you can’t nest aggregates directly (e.g., doing a `COUNT(*)` inside a `JSON_AGG`, `ARRAY_AGG`, etc.). The quick fix is to **compute counts in a subquery**, then aggregate.

### If this was building a `source_mix` blob per market

**Broken pattern (conceptually):**

```sql
SELECT JSON_AGG(JSON_BUILD_OBJECT('source', source, 'n', COUNT(*)))  -- ❌ nested
FROM odds_snapshots
WHERE match_id = $1 AND market = 'h2h'
GROUP BY market;
```

**Safe two-stage pattern (works):**

```sql
WITH counts AS (
  SELECT market, source, COUNT(*) AS n
  FROM odds_snapshots
  WHERE match_id = $1 AND market = 'h2h'
  GROUP BY market, source
)
SELECT market,
       JSONB_OBJECT_AGG(source, n) AS source_mix
FROM counts
GROUP BY market;
```

### If it’s per-book / per-market counts you’re aggregating to JSON

```sql
WITH counts AS (
  SELECT bookmaker_id, market, COUNT(*) AS n
  FROM odds_snapshots
  WHERE match_id = $1
  GROUP BY bookmaker_id, market
)
SELECT JSONB_AGG(JSONB_BUILD_OBJECT('bookmaker_id', bookmaker_id,
                                    'market', market, 'n', n))
FROM counts;
```

Drop this replacement where you assemble the summary right after the APIF insert loop (the log that prints per-match success).

# 1) Quick acceptance checklist (must pass every deploy)

* **Probabilities**: `home_win + draw + away_win ≈ 1.0` (|sum−1| < 1e-9).
* **Argmax alignment**: `recommended_bet` == argmax of the **same** probs you return.
* **Confidence policy**:

  * If `n_triplets < 2` or `time_bucket` missing → `confidence=0`, `recommended_bet="No Bet"`.
  * Else entropy-based confidence reported; min-confidence gate applied (e.g., 0.60).
* **Coverage metadata in response** (so tools/users can debug):
  `time_bucket`, `n_books_raw`, `n_triplets_used`, `has_pinnacle`, `dispersion`, `latest_ts_snapshot`, `low_coverage_flag`.

# 2) Regression tests you should keep (and fail fast on)

* **Simplex test**: probs sum to 1; all ≥ 0; NaN/±inf never appear.
* **Reco alignment**: `recommended_bet` equals `argmax(predictions)`, unless confidence gate forces “No Bet.”
* **Missing-outcome safety**: if any outcome missing for a book snapshot, that **snapshot** is excluded (others still used).
* **Key mapping**: internal (`home/draw/away`) ↔ API surface (`home_win/draw/away_win`) stays consistent.
* **Horizon bucketing**: exact same match at T-72 and T-48 yields **different** results (sanity that you’re not mixing buckets).
* **Database evolution**: if `created_at` disappears or diverges from `ts_snapshot`, tests still use `ts_snapshot`.

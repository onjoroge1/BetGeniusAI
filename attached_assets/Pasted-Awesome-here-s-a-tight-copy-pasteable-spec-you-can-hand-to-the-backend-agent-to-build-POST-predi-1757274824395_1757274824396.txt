2) Semantics / rules
	•	enrich = true when there exists a latest consensus_predictions row for match_id within staleness_hours and it passed gates (your builder already enforces gates; we only check existence).
	•	enrich = false, reason = "waiting_consensus" when we have any odds_snapshots rows for that match_id but no qualifying consensus row yet.
	•	enrich = false, reason = "no_odds" when we have no snapshots at all for that match_id.
	•	bookmakers: if consensus exists, use consensus_predictions.bookmakers; else COUNT(DISTINCT book_id) from odds_snapshots.
	•	last_updated: consensus_predictions.created_at if present, otherwise latest odds_snapshots.ts_snapshot.
	•	min_secs_to_kickoff: MIN(secs_to_kickoff) from odds_snapshots (nullable; handy for FE sorting/badges).

⸻

3) Single SQL (no N+1)

PostgreSQL CTE using array param:

WITH ids AS (
  SELECT UNNEST($1::bigint[]) AS match_id
),
latest_consensus AS (
  SELECT DISTINCT ON (match_id)
         match_id, time_bucket, bookmakers, created_at
  FROM consensus_predictions
  WHERE match_id IN (SELECT match_id FROM ids)
    AND created_at > NOW() - ($2::interval)   -- staleness window
  ORDER BY match_id, created_at DESC
),
odds AS (
  SELECT match_id,
         COUNT(DISTINCT book_id) AS books,
         MAX(ts_snapshot) AS last_snapshot,
         MIN(secs_to_kickoff) AS min_secs_to_kickoff
  FROM odds_snapshots
  WHERE match_id IN (SELECT match_id FROM ids)
  GROUP BY match_id
)
SELECT
  i.match_id,
  (lc.match_id IS NOT NULL) AS enrich,
  CASE
    WHEN lc.match_id IS NOT NULL THEN 'consensus_ready'
    WHEN o.match_id IS NOT NULL THEN 'waiting_consensus'
    ELSE 'no_odds'
  END AS reason,
  lc.time_bucket,
  COALESCE(lc.bookmakers, o.books, 0) AS bookmakers,
  COALESCE(lc.created_at, o.last_snapshot) AS last_updated,
  o.min_secs_to_kickoff
FROM ids i
LEFT JOIN latest_consensus lc USING (match_id)
LEFT JOIN odds o USING (match_id);

Parameters
	•	$1: array of match_ids (deduped, max 100)
	•	$2: interval text like '168 hours' (derived from staleness_hours)

4) Code skeleton (FastAPI-ish)

class AvailabilityReq(BaseModel):
    match_ids: List[int]
    trigger_consensus: bool = False
    staleness_hours: int = 168

@app.post("/predict/availability")
async def predict_availability(req: AvailabilityReq, user=Depends(auth_guard)):
    # Validate
    ids = list(dict.fromkeys(req.match_ids))  # dedupe, preserve order
    if not ids or len(ids) > 100:
        raise HTTPException(400, "match_ids required and must be <= 100")

    rows = await db.fetch("""
        -- (paste SQL above)
    """, ids, f"{req.staleness_hours} hours")

    results = []
    to_trigger = []
    for r in rows:
        enrich = r["enrich"]
        reason = r["reason"]
        if req.trigger_consensus and (not enrich) and reason == "waiting_consensus":
            to_trigger.append(r["match_id"])
        results.append({
            "match_id": r["match_id"],
            "enrich": enrich,
            "reason": reason,
            "time_bucket": r["time_bucket"],
            "bookmakers": r["bookmakers"],
            "last_updated": r["last_updated"].isoformat() if r["last_updated"] else None,
            "min_secs_to_kickoff": r["min_secs_to_kickoff"]
        })

    # Optional: enqueue builder for “waiting_consensus”
    if to_trigger:
        try:
            scheduler.enqueue_consensus_build(to_trigger)  # non-blocking
        except Exception as e:
            logger.warning(f"availability trigger failed: {e}")

    meta = {
        "requested": len(req.match_ids),
        "deduped": len(ids),
        "enrich_true": sum(1 for r in results if r["enrich"]),
        "enrich_false": sum(1 for r in results if not r["enrich"]),
        "failure_breakdown": {
            "no_odds": sum(1 for r in results if r["reason"] == "no_odds"),
            "waiting_consensus": sum(1 for r in results if r["reason"] == "waiting_consensus")
        }
    }
    return {"availability": results, "meta": meta}

8) Validation & limits
	•	Max match_ids per call: 100 (return 400 otherwise).
	•	Ensure all ids are integers; silently drop dupes.
	•	Timeouts: target p95 < 50ms for 100 ids (with the indexes).
	•	Rate-limit per token (e.g., 10 rps) since this endpoint will be called frequently by the FE list view.

10) Tests / acceptance
	•	Case A: consensus exists → enrich=true, reason=consensus_ready, correct bucket/bookmakers.
	•	Case B: snapshots exist, no consensus → enrich=false, reason=waiting_consensus, bookmakers > 0.
	•	Case C: no snapshots → enrich=false, reason=no_odds, bookmakers 0.
	•	Duplicates in input are deduped.
	•	Invalid body (missing/too many ids) returns 400.
	•	With trigger_consensus=true, ensure enqueue is called only for waiting_consensus.

	3.	For others, show “Prediction pending” or “Collecting odds”.

## 2. Feature Engineering Logic

> Implement as reusable Python funcs or SQL views. Use rolling windows and ensure time-ordering.

### 2.1 Team Form (per team, pre-match)

* Rolling last 5 (and maybe 10) matches:

  * Points (Win=3, Draw=1, Loss=0)
  * Goals For/Against averages
  * Goal diff avg
  * Win/Draw/Loss rates
  * Current streak (type + length)
* Home/Away split versions (e.g., `home_form_pts_last5`, `away_form_pts_last5`)

**SQL sketch for last 5 points:**

```sql
WITH ordered AS (
  SELECT m.match_id, m.match_date, t.team_id, 
         CASE 
           WHEN (t.is_home AND m.outcome='H') OR 
                (NOT t.is_home AND m.outcome='A') THEN 3
           WHEN m.outcome='D' THEN 1
           ELSE 0
         END AS pts,
         ROW_NUMBER() OVER (PARTITION BY t.team_id ORDER BY m.match_date) rn
  FROM matches m
  JOIN (
    SELECT match_id, home_team_id AS team_id, TRUE AS is_home FROM matches
    UNION ALL
    SELECT match_id, away_team_id AS team_id, FALSE AS is_home FROM matches
  ) t USING (match_id)
)
SELECT cur.match_id, cur.team_id,
       SUM(prev.pts) AS form_pts_last5
FROM ordered cur
JOIN ordered prev
  ON prev.team_id = cur.team_id
 AND prev.rn     < cur.rn
 AND prev.rn     >= cur.rn - 5
GROUP BY cur.match_id, cur.team_id;
```

### 2.2 Elo Rating

* Initialize each team Elo (e.g., 1500).
* After each match, update using goal difference & outcome.
* Store **elo before match** as feature; compute diff: `elo_home_pre−elo_away_pre`.

### 2.3 Head-to-Head

* Use last N meetings between these two specific clubs.
* Wins/losses, avg goals, etc.
  Only prior to current match.

### 2.4 Market/Odds Features (if captured)

* Convert decimals to implied probs: `imp = 1 / odds`.
* Remove margin: normalize `imp / sum(imp)`.
* Features: differences between model prob and implied prob.

### 2.5 Derived Interaction Features

* `strength_diff = elo_home_pre - elo_away_pre`
* `form_diff = form_pts_last5_home - form_pts_last5_away`
* `draw_tendency = (abs(strength_diff) < small_threshold) AND (low_total_goals_last5_home AND low_total_goals_last5_away)`
* `must_win_flag` late season (table position + points to safety/title)

## 3. Modeling Strategy

### 3.1 Two-Stage Classification (to fix “all draws”)

1. **Model A (Binary):** Predict `Draw` vs `Not-Draw`.
2. **Model B (Binary):** On `Not-Draw` subset, predict `Home` vs `Away`.
3. Combine:

   ```
   P(Draw) = p_draw
   P(Home) = (1 - p_draw) * p_home_given_notdraw
   P(Away) = (1 - p_draw) * (1 - p_home_given_notdraw)
   ```

### 3.2 Algorithms

* Try **CatBoost** (handles categorical + class imbalance well).
* Also try **LightGBM/XGBoost** and compare.
* Keep a **LogisticRegression** baseline.

### 3.3 Class Imbalance Fixes

* Use class weights or focal loss.
* Oversample minority classes WITHIN time folds if needed.

### 3.4 Alternative Target (optional)

* Regress **goal difference** (HomeGoals − AwayGoals).
  Then map to H/D/A using thresholds. Sometimes better.

### 3.5 Calibration

* Apply isotonic or Platt scaling on validation folds for well-calibrated probabilities.

## 4. Validation Protocol (No leakage)

* **Time-based CV (rolling origin):** Train on seasons up to 2019, test 2020, etc.
* **GroupKFold** by season or by match week to avoid overlap.
* Metrics to compute each fold: LogLoss, Brier, Accuracy, Top‑2, Macro-F1, EV.

**Unit tests:**

* Assert no row in training has `match_date >= min(test_match_date)` in that fold.
* Assert no post-match fields leak (goals, outcome, after-the-fact odds).

## 5. Betting Layer & EV Calc

### 5.1 Convert probs to value bets

For each outcome `o`:

* `imp_prob_book = 1 / odds_open_o`
* `edge = model_prob_o - imp_prob_book_adjusted` (remove margin)
* Place bet if `edge >= EDGE_THRESHOLD` (e.g., 3–5%).

### 5.2 Track KPI

* Hit rate of recommended bets
* Average EV per bet
* Overall ROI over rolling windows

## 6. Batch Prediction Workflow (Pre-Match)

**Run daily (e.g., 03:00 UTC):**

1. Pull fixtures for next 72h.
2. Generate features (form, elo, h2h, odds).
3. Load latest model from `models/clean_production_model.joblib`.
4. Predict probabilities.
5. Derive EV vs odds, filter bets, rank by EV/confidence.
6. Save to `predictions` table + push to API/WhatsApp/etc.

**Update `ml_predictor.py`:**

```python
# models/ml_predictor.py
import joblib
class MLPredictor:
    def __init__(self):
        data = joblib.load('models/clean_production_model.joblib')
        self.model_stage1 = data['model_draw_vs_not']
        self.model_stage2 = data['model_home_vs_away']
        self.scaler = data.get('scaler')     # if used
        self.feature_order = data['feature_order']

    def predict(self, feature_df):
        X = feature_df[self.feature_order]
        if self.scaler: X = self.scaler.transform(X)
        p_draw = self.model_stage1.predict_proba(X)[:,1]
        p_home_given_not = self.model_stage2.predict_proba(X)[:,1]
        p_home = (1 - p_draw) * p_home_given_not
        p_away = 1 - p_draw - p_home
        return {'home': p_home, 'draw': p_draw, 'away': p_away}
```

# 2) Why no CLV alerts for 2 days?

**How CLV alerts are generated (simplified):**

* CLV Producer runs every 60s but only triggers when there are **fresh odds (≤10m)** across ≥N books and the fixture is within your watch window.
* If **odds arrive only in 3–6h batches**, there will be long stretches where the 10-minute window is empty → **no alerts** even though the system is “running.”

**Quick investigation (copy/paste):**

```sql
-- A) Were there even any fresh odds in the last 48h windows?
SELECT date_trunc('hour', ts_snapshot) AS hour,
       COUNT(*) AS odds_rows,
       COUNT(DISTINCT match_id) AS matches
FROM odds_snapshots
WHERE ts_snapshot > NOW() - INTERVAL '48 hours'
GROUP BY 1 ORDER BY 1 DESC LIMIT 48;

-- B) For upcoming fixtures (next 72h), do we have fresh odds (<=10m) and >=3 books?
WITH upcoming AS (
  SELECT match_id FROM fixtures
  WHERE kickoff_at BETWEEN NOW() AND NOW() + INTERVAL '72 hours'
),
fresh AS (
  SELECT match_id, COUNT(DISTINCT book_id) AS books
  FROM odds_snapshots
  WHERE ts_snapshot > NOW() - INTERVAL '10 minutes'
  GROUP BY match_id
)
SELECT COUNT(*) AS upcoming,
       COALESCE((SELECT COUNT(*) FROM fresh f JOIN upcoming u USING (match_id) WHERE books >= 3),0) AS matches_meeting_threshold;
```

**Likely outcome given your description:**

* Batch collection ran hours ago; `odds_10m` frequently zero → producer finds nothing.
* In parallel, predictions stalled because consensus isn’t refreshed (next section) which also constrains your downstream metrics.

**Two practical fixes (pick one):**

* **Fast today:** If `odds_10m=0` for ≥15m, trigger a **small seed collection** (top leagues or next 50 fixtures) to re-prime the 10-minute window.
* **Best long-term:** Implement **targeted per-match collection** in Phase B (bounded concurrency). Then the 10-minute window almost never empties, and CLV fires continuously.

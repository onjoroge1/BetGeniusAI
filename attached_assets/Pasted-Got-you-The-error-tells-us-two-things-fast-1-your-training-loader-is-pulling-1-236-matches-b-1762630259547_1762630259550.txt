# 2) Provide BOTH “latest pre-KO” and “opening” snapshots

Your 50-feature builder computes drift/volatility/dispersion using both last and opening probabilities. If you only built a single `odds_real_consensus` (“latest near T-1h”), drift features will be NaN and your builder might reject the row.

Create two materialized views from `odds_snapshots`:

```sql
-- LATEST pre-KO (nearest to 60m before KO, >=5m pre-KO)
CREATE MATERIALIZED VIEW IF NOT EXISTS odds_real_latest_preko AS
WITH c AS (
  SELECT
    match_id, book_id, outcome,
    implied_prob,
    ROW_NUMBER() OVER (
      PARTITION BY match_id, book_id, outcome
      ORDER BY ABS(secs_to_kickoff - 3600)
    ) rn
  FROM odds_snapshots
  WHERE market='1X2' AND odds_decimal>1.01 AND secs_to_kickoff>300
)
SELECT match_id,
       COUNT(DISTINCT book_id) AS n_books,
       AVG(CASE WHEN outcome='home' THEN implied_prob END) AS p_home,
       AVG(CASE WHEN outcome='draw' THEN implied_prob END) AS p_draw,
       AVG(CASE WHEN outcome='away' THEN implied_prob END) AS p_away
FROM c
WHERE rn=1
GROUP BY match_id
HAVING COUNT(DISTINCT book_id) >= 3;

-- OPENING snapshot (earliest we have; you can tighten the window later)
CREATE MATERIALIZED VIEW IF NOT EXISTS odds_real_opening AS
WITH c AS (
  SELECT
    match_id, book_id, outcome,
    implied_prob,
    ROW_NUMBER() OVER (
      PARTITION BY match_id, book_id, outcome
      ORDER BY secs_to_kickoff DESC   -- farthest from KO = earliest
    ) rn
  FROM odds_snapshots
  WHERE market='1X2' AND odds_decimal>1.01 AND secs_to_kickoff>300
)
SELECT match_id,
       COUNT(DISTINCT book_id) AS n_books,
       AVG(CASE WHEN outcome='home' THEN implied_prob END) AS p_home_open,
       AVG(CASE WHEN outcome='draw' THEN implied_prob END) AS p_draw_open,
       AVG(CASE WHEN outcome='away' THEN implied_prob END) AS p_away_open
FROM c
WHERE rn=1
GROUP BY match_id
HAVING COUNT(DISTINCT book_id) >= 3;

-- (Optional convenience view that merges both)
CREATE MATERIALIZED VIEW IF NOT EXISTS odds_real_consensus AS
SELECT
  l.match_id,
  l.n_books,
  l.p_home, l.p_draw, l.p_away,
  o.p_home_open, o.p_draw_open, o.p_away_open
FROM odds_real_latest_preko l
LEFT JOIN odds_real_opening o USING (match_id);
```

Your corruption post-mortem and clean-data plan match this approach (use odds_snapshots; enforce pre-KO and n_books; never touch odds_consensus).

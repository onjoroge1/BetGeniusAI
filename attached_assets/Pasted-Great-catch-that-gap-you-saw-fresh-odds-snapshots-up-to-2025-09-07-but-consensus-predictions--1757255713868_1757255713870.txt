# 1) Quick health checks (paste into psql)

**Freshness + gap**

```sql
-- Latest timestamps + how many matches still lack consensus
WITH last_cons AS (SELECT MAX(created_at) AS t FROM consensus_predictions),
     last_odds AS (SELECT MAX(ts_snapshot) AS t FROM odds_snapshots)
SELECT
  (SELECT t FROM last_odds)       AS latest_odds_snapshot,
  (SELECT t FROM last_cons)       AS latest_consensus,
  (SELECT (SELECT t FROM last_odds) - (SELECT t FROM last_cons)) AS time_gap,
  (SELECT COUNT(DISTINCT o.match_id)
     FROM odds_snapshots o
     LEFT JOIN consensus_predictions c ON c.match_id = o.match_id
    WHERE o.ts_snapshot > (SELECT t FROM last_cons)
      AND c.match_id IS NULL)     AS matches_missing_consensus;
```

**24-hour coverage**

```sql
WITH o AS (
  SELECT DISTINCT match_id FROM odds_snapshots WHERE ts_snapshot > NOW() - INTERVAL '24 hours'
),
c AS (
  SELECT DISTINCT match_id FROM consensus_predictions WHERE created_at > NOW() - INTERVAL '24 hours'
)
SELECT
  (SELECT COUNT(*) FROM o) AS matches_with_odds_24h,
  (SELECT COUNT(*) FROM c) AS matches_with_consensus_24h,
  ROUND((SELECT COUNT(*) FROM c)*100.0/NULLIF((SELECT COUNT(*) FROM o),0),1) AS coverage_pct;
```

**Bucket distribution (to confirm the fixed ranges are populating)**

```sql
SELECT time_bucket, COUNT(*) AS rows, COUNT(DISTINCT match_id) AS matches
FROM consensus_predictions
WHERE created_at > NOW() - INTERVAL '48 hours'
GROUP BY time_bucket
ORDER BY CASE time_bucket
  WHEN '3h' THEN 1 WHEN '6h' THEN 2 WHEN '12h' THEN 3
  WHEN '24h' THEN 4 WHEN '48h' THEN 5 WHEN '72h' THEN 6 ELSE 7 END;
```

✅ You’re healthy if:

* `time_gap` is small (≤ 1–2 hours) and **matches\_missing\_consensus = 0–2**.
* `coverage_pct ≥ 80%` for the last 24h.
* Buckets show rows across your active windows (12/24/48/72h).

# What’s happening

* After the resolver finds a fixture via **Step 3 (live search)**, we upsert the row into `fixtures`.
* That upsert path builds a UTC datetime like:

  ```python
  kickoff_at_utc = datetime.fromtimestamp(ts, tz=timezone.utc)
  ```
* But `timezone` wasn’t imported → **`NameError: name 'timezone' is not defined`**.
* The upsert fails, we warn, and then we still insert odds rows → this can re-introduce **orphaned odds** (odds in `odds_snapshots` whose `match_id` isn’t in `fixtures`).

# Minimal hotfix (do this now)

In `utils/api_football_integration.py`:

1. Fix imports at the top:

```python
from datetime import datetime, timezone  # add timezone
# or: import datetime as dt  and then use dt.timezone.utc below
```

2. Use the imported symbol consistently (two safe patterns):

**Pattern A**

```python
kickoff_at_utc = datetime.fromtimestamp(ts, tz=timezone.utc)
```

**Pattern B**

```python
import datetime as dt
kickoff_at_utc = dt.datetime.fromtimestamp(ts, tz=dt.timezone.utc)
```

3. (Optional but recommended) Add a tiny fallback so a missing import never blocks fixture upsert again:

```python
try:
    kickoff_at_utc = datetime.fromtimestamp(ts, tz=timezone.utc)
except NameError:
    from datetime import timezone as _tz
    kickoff_at_utc = datetime.fromtimestamp(ts, tz=_tz.utc)
```

# Safety rail while you’re here

Wrap the **fixture upsert** in its own savepoint so it can’t be skipped if odds insert later fails:

```python
cur.execute("SAVEPOINT sp_fixture")
try:
    cur.execute("""INSERT INTO fixtures (match_id, league_id, home_team, away_team, kickoff_at, season)
                   VALUES (%s,%s,%s,%s,%s,%s)
                   ON CONFLICT (match_id) DO UPDATE
                   SET league_id=EXCLUDED.league_id,
                       home_team=EXCLUDED.home_team,
                       away_team=EXCLUDED.away_team,
                       kickoff_at=EXCLUDED.kickoff_at,
                       season=EXCLUDED.season""",
                (match_id, league_id, home, away, kickoff_at_utc, season))
    cur.execute("RELEASE SAVEPOINT sp_fixture")
except Exception as e:
    logger.warning(f"Failed to upsert fixture {match_id}: {e}")
    cur.execute("ROLLBACK TO SAVEPOINT sp_fixture")
```

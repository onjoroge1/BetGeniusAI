## PRIORITY 1 — Fix the weighted-consensus application (blocker)

**Symptom:** 100% identical `weighted == equal` indicates the model is still using a median or unweighted mean, or it’s falling back incorrectly when weights exist.

### A. Implement the actual weighted consensus (per match, per outcome)

For each match snapshot at horizon **H** (e.g., T-72):

1. Convert each book’s H/D/A decimal odds → **margin-adjusted implied probabilities** $q^{(b)}$.
2. Read **era×league weights** $w_b$ (non-negative; normalized on **available** books only).
3. Compute weighted consensus:

$$
q^{\text{wt}}_o \;=\; \frac{\sum_{b \in \mathcal{B}_\text{avail}} w_b \, q^{(b)}_o}{\sum_{b \in \mathcal{B}_\text{avail}} w_b},\quad
o \in \{H,D,A\}
$$

4. Re-normalize $q^{\text{wt}}$ to sum to 1.

**Edge cases / fallbacks**

* If fewer than **2** available books → **equal-weight** over the available books (and flag `consensus_fallback=1`).
* If no weights for a book’s era/league → use the nearest era; if still missing → equal-weight fallback with flag.

### B. Store what you applied (for auditability)

Augment `odds_consensus` with:

* `consensus_method` (e.g., `weighted_v1`),
* `weights_applied_json` (map `{book_id: effective_weight}` for that match),
* `consensus_fallback` (0/1), `n_books_used`.

### C. Acceptance tests (must pass)

* **Consensus QA** (`consensus_qa.py`):

  * `identical_triplet_pct` **< 10%** overall when `n_books ≥ 2`.
  * Mean |weighted–equal| **> 0** in all five Euro leagues.
  * Split by `has_pinnacle`: diffs should be **larger** when sharp book is present.
* **Unit test**: for a synthetic row with two books and **distinct weights**, assert `weighted != equal` (within 1e-9).

**Expected impact:** once actually applied, **weighted consensus** typically buys **−0.005 to −0.010 LogLoss** vs equal-weight Market-T72 in major leagues.

## PRIORITY 2 — Instance-wise book mixing (learn weights per match)

Static era tables are a good start; **learning per-match weights** is better.

### A. Meta-learner over book logits (constrained, interpretable)

* **Inputs:** for each match, stack each available book’s **logits** $\log(q^{(b)})$, plus context features: `dispersion`, `n_books`, overround, league id, era bin, season phase; optionally book coverage indicators.
* **Model options:**

  1. **Constrained multinomial LR**: non-negative coefficients on book logits + L1 (sparsify).
  2. **Softmax gating network**: compute $w_b(x) = \text{softmax}(h(x))$ with $h(x)$ a small linear model; weights are ≥0 and sum to 1 by construction.
* **Output:** per-match blended prior $q^{\text{mix}}$. Use it **instead of** static weighted $q^{\text{wt}}$ when sufficient books exist.

**Acceptance:** show ΔLogLoss of **mixing** vs **static weighted** on the same offline slice with paired bootstrap CIs. If CI excludes 0 and ΔLL ≤ −0.003 overall, keep it; else fall back to static weights.

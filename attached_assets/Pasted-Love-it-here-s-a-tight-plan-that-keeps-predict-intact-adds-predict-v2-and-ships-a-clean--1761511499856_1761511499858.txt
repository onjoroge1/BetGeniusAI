# How we compute/confidence & selection

* `v1_probs` = no-vig probabilities from **current** odds (or your consensus table).
* `v2_probs` = LightGBM V2 prediction (pre-match; can be recomputed when inputs change).
* `conf_v2 = max(v2_probs.values())`.
* `ev_live = max(v2_probs) − max(v1_probs_current)`.
* **V2-Select gate** (premium): `conf_v2 ≥ 0.62 and ev_live > 0 and league_ece ≤ 0.05`.

If V1 is missing for a match, leave the V1 fields **null** (don’t default to 33%). The UI will hide the V1 row for that match.

# API contracts

### 1) `/predict-v2` (V2-Select)

* Works like `/predict`, but:

  * swaps V1 for V2,
  * enforces the **Select** gate,
  * (optionally) calls your OpenAI analyzer for reasons.
* **Available listings**: reuse `/predict/available?model=v2` (same handler adds a filter for Select-eligible matches).

**Request**

```http
POST /predict-v2
{
  "match_id": "af:fixture:12345",
  "include_analysis": true
}
```

**Response** (same shape as `/predict`)

```json
{
  "match_info": { ... },
  "predictions": {
    "home_win": 0.21, "draw": 0.18, "away_win": 0.61,
    "confidence": 0.61,

    "confidence_threshold": 0.62

```

If the match doesn’t qualify for Select, return **403** with a helpful reason and a CTA to use `/predict` (V1) or view `/market`.

# Cadence & data freshness

* **Collectors** keep `odds_snapshots` fresh (e.g., 30–60s upcoming; 5–10s live).
* `/market` is **read-only** with a 5–10s Redis cache TTL.
* A tiny **picker job** (same cadence) can pre-compute Select eligibility and cache:

  * `conf_v2`, `ev_live`, `premium_lock`, and a compact UI blob.

# Handling edge cases (important)

* **No V1 odds** yet → set `models.v1_consensus.probs = null` (UI hides V1 row).
* **No V2** (rare—e.g., data gap) → set `models.v2_lightgbm.probs = null` and `ui_hints.primary_model = "v1_consensus"`.
* **Books disagree** → still compute no-vig from available books; expose per-book odds as in the mock.
* **Live** → pull score/clock from API-Football; same structure.

# Minimal SQL for latest odds (per book, per match)

```sql
WITH latest AS (
  SELECT DISTINCT ON (match_id, bookmaker, outcome)
    match_id, bookmaker, outcome, odds_decimal
  FROM odds_snapshots
  WHERE match_id = $1 AND market = 'h2h'
  ORDER BY match_id, bookmaker, outcome, ts_snapshot DESC
)
SELECT bookmaker,
       MAX(CASE WHEN outcome='H' THEN odds_decimal END) AS home,
       MAX(CASE WHEN outcome='D' THEN odds_decimal END) AS draw,
       MAX(CASE WHEN outcome='A' THEN odds_decimal END) AS away
FROM latest
GROUP BY bookmaker;
```

Then convert to no-vig probabilities for `novig_current`.

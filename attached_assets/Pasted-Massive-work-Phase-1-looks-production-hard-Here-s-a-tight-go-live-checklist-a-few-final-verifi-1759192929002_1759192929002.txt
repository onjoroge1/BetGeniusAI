Final smoke tests (run these today)

1) Dedup sanity (desk groups)

-- Should equal number of DISTINCT desk_group we actually ingest
SELECT COUNT(DISTINCT desk_group) AS groups, COUNT(*) AS books FROM bookmakers;

Ensure books_used in API equals the groups count used by the producer for each alert.

2) De-juice identity
Add an assert in your engine (or log once per cycle): abs(pH+pD+pA-1.0) < 1e-9. If it ever fires, suppress with BAD_IDENTITY.

3) TTL + uniqueness
	•	Insert same opportunity twice in a minute → confirm second insert is skipped (unique constraint).
	•	Expire > 1h → confirm TTL cleanup removes rows (check clv_alerts count before/after).

4) Stability gate
Feed a synthetic volatile series (std≈0.05): stability should drop <0.70 and produce LOW_STABILITY suppressions in logs.

5) API contract
GET /clv/club/opportunities should include:

{"books_used":10,"stability":0.83,"window":"T-48to24",
 "composite_method":"trim_10_dejuiced","closing_method":"LAST5_VWAP"}

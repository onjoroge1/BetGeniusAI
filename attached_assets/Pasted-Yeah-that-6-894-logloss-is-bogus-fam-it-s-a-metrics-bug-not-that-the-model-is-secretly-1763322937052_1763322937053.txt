## 2️⃣ Why the OOF logloss is so high (and why that warning shows up)

From `train_v2_transformed.py`, your OOF block does:

```python
oof_proba = np.zeros((len(df), 3))
oof_true = y.copy()

for fold, (tr_idx, va_idx) in enumerate(cv.split(X, y, groups=dates), start=1):
    ...
    proba_va = model.predict(X_va, num_iteration=model.best_iteration)
    oof_proba[va_idx] = proba_va
    ...

overall_acc = accuracy_score(oof_true, np.argmax(oof_proba, axis=1))
overall_ll  = log_loss(oof_true, oof_proba)
one_hot     = np.eye(3)[oof_true]
overall_br  = np.mean(np.sum((oof_proba - one_hot)**2, axis=1)) / 2.0
```

Key problem: **some rows in `oof_proba` are never filled.**

Because you’re using a **purged time-series split with an embargo window**, some samples will:

* Never land in any validation split (they only appear in training parts).
* So their row in `oof_proba` stays as the initial **[0, 0, 0]**.

Then you call:

```python
log_loss(oof_true, oof_proba)
```

on the whole array. For any row where the true class is `k` and `p_k = 0`, the log loss blows up:

* log(0) → −∞ (sklearn clips, but it becomes a huge penalty),
* Sum across all those “empty” rows → **massive average logloss** → 6.894.

That also explains the warning:

> “The y_prob values do not sum to one.”

Because those [0,0,0] rows obviously don’t sum to 1.

👉 So the **true performance** is in the **per-fold logloss (~1.0)**,
and the **6.894** is just the OOF metric polluted by unfilled rows.

---

## 3️⃣ How to fix it (minimal code change)

Easiest fix: **only compute OOF metrics on rows that actually got validation predictions.**

Inside your CV loop, track which indices were used:

```python
oof_proba = np.zeros((len(df), 3))
oof_true  = y.copy()
used_mask = np.zeros(len(df), dtype=bool)

for fold, (tr_idx, va_idx) in enumerate(cv.split(X, y, groups=dates), start=1):
    ...
    proba_va = model.predict(X_va, num_iteration=model.best_iteration)
    oof_proba[va_idx] = proba_va
    used_mask[va_idx] = True
    ...
```

Then, for OOF metrics:

```python
mask = used_mask

overall_acc = accuracy_score(oof_true[mask],
                             np.argmax(oof_proba[mask], axis=1))
overall_ll  = log_loss(oof_true[mask], oof_proba[mask])

one_hot = np.eye(3)[oof_true[mask]]
overall_br = np.mean(np.sum((oof_proba[mask] - one_hot)**2, axis=1)) / 2.0
```

That will:

* Remove the [0,0,0] rows from the metric calculation,
* Kill the sklearn “probabilities do not sum to one” warning,
* Bring OOF logloss down to something close to your fold values (~1.0–1.1).

If you want to be fancy, you could instead fill unused rows in `oof_proba` with the **class priors** or a global model prediction, but honestly, just masking to “seen in some validation fold” is the cleanest.

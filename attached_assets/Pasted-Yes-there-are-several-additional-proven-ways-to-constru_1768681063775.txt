## 1) Slate-level “Best Bets First” + Parlay as Packaging

**Concept:** Do not “build parlays.” Instead, **rank single legs** by quality, then package the top legs into 2-leg parlays.

**Mechanics**

* Compute a **Leg Quality Score** per candidate bet:

  * Base: `EV = p*odds - 1`
  * Penalize longshots: `- λ * max(0, odds - 3.0)`
  * Penalize low probability: `- μ * max(0, p_min - p)`
  * Penalize model uncertainty (if you have it): `- ν * σ`
* Take the **top N legs across the slate** (from different matches)
* Create parlays using best combinations subject to:

  * different matches
  * low correlation
  * target win-prob band (e.g., 18–30%)

**Why it’s high quality**

* Forces diversity across matches
* Eliminates “one match spam”
* Ensures every parlay is composed of legs that would be good as singles

---

## 2) Probability-Banded Parlays (Product-grade “tiers”)

**Concept:** Create parlays by **target hit rate**, not by odds.

**Bands (example)**

* **Safe 2-leg:** parlay win prob ≥ 25% and each leg p ≥ 60%
* **Standard 2-leg:** parlay win prob 18–25% and each leg p ≥ 55%
* **Value 2-leg:** parlay win prob 12–18% and each leg p ≥ 50%

**Why it works**

* This is the simplest way to align “confidence” with reality.
* Users understand it immediately.
* It prevents the model from pushing 5% parlays as “high.”

## 3) “Same-Game Parlay” Only When Correlation Is Positive and Modeled

Right now you are applying a correlation penalty, but the construction still behaves like “SGP for fun.” A high-quality approach is:

**Only allow same-match parlays when:**

* you are intentionally exploiting **positive correlation**, AND
* you have a correlation model / heuristic that is directionally correct.

**Examples of coherent, positively correlated combos**

* Home Win + Over 1.5 (home dominance + goals)
* Draw + Under 2.5 (tight match)
* Underdog + Under 3.5 (scrappy/low event)
* BTTS Yes + Over 2.5 (goals environment)

**What to avoid**

* Home Win + Under 1.5 (rare scorelines, fragile)
* Away longshot + random total added as filler

**Implementation note**
If you can’t properly model correlation yet, keep SGP as a distinct category and label it as such; do not mix it into “Trust Parlays.”

## 4) Portfolio Construction: Maximize Expected Log Utility Under Risk Constraints

This is a professional approach.

**Concept:** You treat each parlay as an “asset” with:

* expected return (EV)
* variance (from probability)
* covariance (from match overlap / league / style correlation)

Then you build a “portfolio” of parlays that maximizes expected utility subject to constraints:

* exposure caps per match/team/league
* probability floor
* odds ceiling
* diversity requirements

**Practical lightweight version**

* Use **greedy selection**:

  * Start from best parlay score
  * Add next only if it doesn’t violate exposure and doesn’t overlap too much
  * Continue until you have the daily set

This yields a feed that feels curated and “smart.”

## 5) “Consensus + Edge” Hybrid (Market-aware filtering)

Right now you can get fake “edge” because of calibration quirks on long odds. A clean fix:

**Require BOTH:**

* model edge positive, AND
* market sanity checks:

  * de-vig implied prob not wildly far from model prob unless strong evidence
  * exclude outcomes where the model disagrees by an extreme amount *and* the outcome is long-odds

**Simple rule**

* If `odds > 4.0`, require `p_model >= 0.30` AND `edge >= +5%`
* If `odds > 8.0`, require `p_model >= 0.20` AND `edge >= +8%` AND `match_quality >= threshold`

This prevents the “9.4 odds away win” from dominating the feed unless it truly qualifies.

## 1. Is ~50–51% accuracy “the best we can get”?

### Short answer

**For a clean, leakage-free, pre-match 3-class (H/D/A) model using odds-centric features: yes, you are already near the realistic ceiling for *accuracy*.**

### Why this is not a failure

Football is:

* Low-scoring
* High-variance
* Draw-heavy
* Structurally asymmetric (home bias, tactical game states)

Even **bookmakers themselves** struggle to exceed:

* **52–54% top-1 accuracy** on H/D/A *after margin removal*
* They compete on **calibration, margin control, and market timing**, not raw hit rate

Your *real* win is here:

| Metric                          | Why it matters                                      |
| ------------------------------- | --------------------------------------------------- |
| **LogLoss ↓ 2.2%**              | Indicates *better probability estimation*, not luck |
| **Stable best iteration (~63)** | Indicates reduced overfitting                       |
| **Pinnacle odds dominance**     | Confirms market-aligned signal                      |
| **Derived ratios ranking high** | Confirms non-linear structure is being learned      |

If this were a Kaggle-style dataset, this is where most strong baselines land.

## 2. Should you move to 2-way / pairwise models and combine them?

### Yes — **but not the way most people do it**

What you are describing is **pairwise decomposition**, and it *does* work in football **if done correctly**.

### The correct framing (important)

Instead of:

> One 3-class softmax model

Move to:

> **Three calibrated binary experts**, each answering a *specific question*

### Recommended binary tasks

| Binary Model         | Target       |
| -------------------- | ------------ |
| **Home vs Not-Home** | H vs (D + A) |
| **Away vs Not-Away** | A vs (H + D) |
| **Draw vs Not-Draw** | D vs (H + A) |

These are *not symmetric problems* — draw prediction especially behaves very differently.

### Why this helps

* Draws are **structurally under-learned** in 3-class softmax
* LightGBM handles **binary class imbalance** better than multinomial entropy
* Feature importance differs by task

  * Draws like *competitiveness & agreement*
  * Wins like *favorite confidence & spread*

---

## 3. How to recombine them (this is the key part)

### Step 1: Train each binary model independently

Each model outputs:

```
P_home_win_binary
P_away_win_binary
P_draw_binary
```

Each one **individually calibrated** (Platt or isotonic).

---

### Step 2: Normalize into a coherent H/D/A distribution

One robust approach:

```text
raw_H = P_home
raw_A = P_away
raw_D = P_draw * draw_boost_factor
```

Then:

```text
sum = raw_H + raw_A + raw_D

P(H) = raw_H / sum
P(D) = raw_D / sum
P(A) = raw_A / sum
```

Where:

* `draw_boost_factor` ≈ 1.05–1.15 (learned on validation)

This avoids the common pitfall where draw probability collapses.

---

## 4. Should these pairwise outputs become *features* instead?

### Yes — this is actually the highest-ROI option

Your best next step is **NOT replacing V2**, but **stacking on top of it**.

### Architecture I recommend

**Layer 1 (experts):**

* Binary H vs not-H model
* Binary A vs not-A model
* Binary D vs not-D model

**Layer 2 (meta model):**

* LightGBM (or logistic) that takes:

  * Original odds features
  * All derived features you added
  * **The 3 binary probabilities**
  * Optional interactions (e.g. P_home − P_away)

This gives you:

* Better calibration
* Better separation of close matches
* A *decision surface* that softmax alone cannot learn

This is how betting syndicates structure ensemble logic.

## 5. Feature engineering directions that still have headroom

You are largely **odds-saturated** now. Remaining gains come from *structure*, not more odds.

### High-leverage feature classes still missing

#### A. Market shape (not just price)

* Odds *curvature* across books
* Favorite skew asymmetry
* Draw compression signals

#### B. Temporal market movement (pre-kickoff only)

* Open → close drift
* Pinnacle vs soft-book divergence
* Late draw tightening

#### C. Regime features

* League-level draw rate deviation (rolling)
* Team-specific draw propensity vs market
* Home-favorite fragility indicator

These help mostly with **draw calibration**, your hardest class.

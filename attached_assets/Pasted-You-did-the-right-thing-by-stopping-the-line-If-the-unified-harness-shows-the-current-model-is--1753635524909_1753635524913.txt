### R3) Fix Top‑2 metric bug (quick)

Top‑2 should check whether the **true class is among the two largest predicted probs**:

```python
def top2_acc(y_true, proba, labels=('H','D','A')):
    import numpy as np
    top2 = np.argsort(-proba, axis=1)[:, :2]          # indices of two highest probs
    true_idx = np.array([labels.index(y) for y in y_true])
    return (top2[:,0] == true_idx) | (top2[:,1] == true_idx)
```

Expect **Top‑2 ≥ 90–95%** for a reasonable forecaster. (Your earlier 72–79% was a red flag.)

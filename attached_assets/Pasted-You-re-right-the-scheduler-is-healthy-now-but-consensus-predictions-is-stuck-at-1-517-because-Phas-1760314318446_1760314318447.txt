1) Target selection (SQL you can paste)

Choose matches that actually need predictions — not all leagues.

-- A. Upcoming fixtures that should have predictions now (window tunable)
WITH upcoming AS (
  SELECT f.match_id
  FROM fixtures f
  WHERE f.kickoff_at BETWEEN NOW() + INTERVAL '6 hours' AND NOW() + INTERVAL '168 hours'
),

-- B. Have fresh odds in last 10m?  (so consensus can be meaningful)
fresh_odds AS (
  SELECT DISTINCT os.match_id
  FROM odds_snapshots os
  WHERE os.ts_snapshot > NOW() - INTERVAL '10 minutes'
),

-- C. Missing or stale predictions (older than 30m or none)
stale_preds AS (
  SELECT u.match_id
  FROM upcoming u
  LEFT JOIN LATERAL (
    SELECT MAX(cp.created_at) AS last_pred_at
    FROM consensus_predictions cp
    WHERE cp.match_id = u.match_id
  ) p ON TRUE
  WHERE p.last_pred_at IS NULL OR p.last_pred_at < NOW() - INTERVAL '30 minutes'
)

SELECT sp.match_id
FROM stale_preds sp
JOIN fresh_odds fo USING (match_id)
ORDER BY sp.match_id
LIMIT 50;  -- bound per tick

This yields a small queue of match_ids that have fresh odds and either no predictions yet or stale ones.

2) Non-blocking Phase B (spawned task) that builds predictions

Wire this after odds collection (or even if collection returns 0, we can still build from existing fresh odds):

# utils/scheduler.py (inside your non-blocking Phase B task)
async def _run_phase_b_fresh_odds(self):
    # 1) (optional) collect fresh odds for target fixtures only
    targets = await db.fetch("/* SQL above */")
    match_ids = [r["match_id"] for r in targets]

    if match_ids:
        await self.collector.collect_for_matches(match_ids, concurrency=6, timeout_per_call=8)

    # 2) ALWAYS attempt to (re)build predictions for the same targets
    #    …even if step 1 fetched 0, because fresh odds may already be present.

    logger.info(f"📈 consensus_predictions: built={built} for {len(match_ids)} targets")

Notes
	•	collect_for_matches() should accept a list of match_ids and do bounded concurrency (e.g., 6) so Phase B stays <30s.
	•	If you don’t have collect_for_matches() yet, temporarily skip fetching and just run the builder — it will use existing fresh odds (odds_snapshots).

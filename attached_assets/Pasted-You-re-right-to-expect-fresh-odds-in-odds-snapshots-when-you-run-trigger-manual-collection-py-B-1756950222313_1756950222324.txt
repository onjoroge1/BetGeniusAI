# 1) One-time fixes to lock in

# 3) What to watch if it’s still empty

* **Wrong league mapping** → API returns zero events for the sport key. Fix mapping.
* **Name mismatch** → see logs for “near-miss” matches (0.85–0.92). Add aliases.
* **Bucket miss** → kickoff time sits outside tolerances. Loosen to suggested bounds.
* **Region/market filter** → ensure you request `markets=h2h` and `regions=eu` (or your preferred regions list).
* **Odds API limits** → check `x-requests-remaining` header. If 0, pause/retry later.
* **DB conflict** → primary key is `(match_id, book_id, ts_snapshot, outcome)`. If you upsert incorrectly, nothing lands. Ensure `ON CONFLICT DO UPDATE` by **newer** `ts_snapshot`.

# 7) Quick SQL to see you’re getting the 72/48/24 streams

```sql
SELECT time_bucket,
       COUNT(DISTINCT match_id) matches,
       ROUND(AVG(n_books),2) avg_books,
       ROUND(AVG(n_triplets),2) avg_triplets
FROM (
  SELECT match_id,
         CASE
           WHEN secs_to_kickoff BETWEEN 36*3600 AND 60*3600 THEN '48h'
           WHEN secs_to_kickoff BETWEEN 18*3600 AND 30*3600 THEN '24h'
           WHEN secs_to_kickoff BETWEEN 66*3600 AND 78*3600 THEN '72h'
           ELSE 'other'
         END AS time_bucket,
         COUNT(DISTINCT book_id) AS n_books,
         COUNT(*) FILTER (WHERE outcome IN ('H','D','A'))/3 AS n_triplets
  FROM odds_snapshots
  WHERE ts_snapshot > NOW() - INTERVAL '24 hours'
  GROUP BY match_id, time_bucket
) t
GROUP BY time_bucket
ORDER BY time_bucket;
```

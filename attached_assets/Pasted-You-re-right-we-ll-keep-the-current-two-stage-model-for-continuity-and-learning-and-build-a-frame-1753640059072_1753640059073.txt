2) Database: Minimal DDL to Scale to 10k+

Matches (canonical)

CREATE TABLE IF NOT EXISTS matches (
  match_id        BIGINT PRIMARY KEY,
  league_id       INT NOT NULL,
  season          INT NOT NULL,
  match_date_utc  TIMESTAMP NOT NULL,
  venue           VARCHAR(200),
  neutral_flag    BOOLEAN DEFAULT FALSE,
  home_team_id    INT NOT NULL,
  away_team_id    INT NOT NULL,
  outcome         CHAR(1) CHECK (outcome IN ('H','D','A')),
  home_goals      INT,
  away_goals      INT
);
CREATE INDEX IF NOT EXISTS idx_matches_league_season ON matches(league_id, season);
CREATE INDEX IF NOT EXISTS idx_matches_date ON matches(match_date_utc);

Odds snapshots (multi-book, multi-time)

CREATE TABLE IF NOT EXISTS odds_snapshots (
  match_id        BIGINT,
  league_id       INT,
  book_id         VARCHAR(32),
  ts_snapshot     TIMESTAMP,
  secs_to_kickoff INT,
  outcome         CHAR(1) CHECK (outcome IN ('H','D','A')),
  odds_decimal    DOUBLE PRECISION,
  implied_prob    DOUBLE PRECISION,
  market_margin   DOUBLE PRECISION,
  PRIMARY KEY(match_id, book_id, ts_snapshot, outcome)
);
CREATE INDEX IF NOT EXISTS idx_snapshots_match ON odds_snapshots(match_id, ts_snapshot);

Feature store (team-match features; two rows per match—home & away—then pivot/merge)

CREATE TABLE IF NOT EXISTS team_features (
  match_id BIGINT,
  team_id  INT,
  is_home  BOOLEAN,
  form_pts_last5 INT,
  gf_avg5 DOUBLE PRECISION,
  ga_avg5 DOUBLE PRECISION,
  elo_pre DOUBLE PRECISION,
  elo_trend5 DOUBLE PRECISION,
  rest_days INT,
  h2h_wins5 INT,
  h2h_draws5 INT,
  h2h_goal_diff5 DOUBLE PRECISION,
  season_phase VARCHAR(16), -- early/mid/late
  manager_tenure_days INT,
  PRIMARY KEY(match_id, team_id, is_home)
);

5) Calibrated-Consensus Forecaster (Production)

Consensus build
	•	At a given prediction time bucket (e.g., 24–6h before KO), for each match:
	•	Collect all books’ implied probs (vig-removed).
	•	Compute median (or trimmed mean) per outcome → q_consensus.
	•	Compute dispersion (std or IQR) per outcome for context.

Per-league, per-bucket calibration

python -m src.consensus.calibrate_consensus \
  --buckets "24h,6h" --leagues "EPL,LALIGA,SERIEA,BUNDES,LIGUE1"

	•	Fit isotonic (or Dirichlet) on OOF predictions only.

7) Promotion Gate (for any residual/two-stage model)

A research model can be promoted only if ALL pass:
	•	Per league: LogLoss(model_calibrated) ≤ LogLoss(market_implied) - 0.005
	•	Overall: Top-2 ≥ 95%, Brier ≤ 0.205
	•	Stability: Improvements reproducible across two consecutive weekly runs
	•	Parity: API inference == offline within 1e-6 on a frozen test batch
	•	No leakage & feature contract tests green

If not, keep it in shadow and stick to Calibrated-Consensus for users.

2) Make autoscale detection bulletproof (don’t depend on TYPE)

Right now your logs show deployment=True, type=unknown and then the scheduler starts. Let’s hard-disable background tasks for any deploy unless explicitly overridden.

In main.py (or your env helper), change the guards to this:

import os

def is_deploy() -> bool:
    # Replit sets REPLIT_DEPLOYMENT=1 for both Autoscale and Reserved VM.
    return os.getenv("REPLIT_DEPLOYMENT", "") == "1"

def bg_enabled() -> bool:
    # In ANY deployment, default to OFF unless you explicitly force it on.
    if is_deploy():
        return os.getenv("ENABLE_BACKGROUND", "0") == "1"
    # In dev, default to ON unless explicitly off.
    return os.getenv("ENABLE_BACKGROUND", "1") == "1"

And in your startup/lifespan, never start the scheduler if bg_enabled() is False:

@app.on_event("startup")
async def on_startup():
    if not bg_enabled():
        logger.info("🚫 DEPLOYMENT MODE: background tasks DISABLED (API-only)")
        return
    asyncio.get_event_loop().create_task(_start_background_jobs())

That guarantees you won’t see “✅ Background scheduler started” on Autoscale unless you set ENABLE_BACKGROUND=1 on purpose.

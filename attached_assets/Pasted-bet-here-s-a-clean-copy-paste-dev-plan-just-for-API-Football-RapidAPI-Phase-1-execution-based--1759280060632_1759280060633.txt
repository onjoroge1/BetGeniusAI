1) Minimal SQL (idempotent, no PK change)

-- A) add source + vendor ids (traceability)
ALTER TABLE odds_snapshots
  ADD COLUMN IF NOT EXISTS source TEXT DEFAULT 'theodds',
  ADD COLUMN IF NOT EXISTS vendor_fixture_id BIGINT,
  ADD COLUMN IF NOT EXISTS vendor_book_id TEXT;

-- B) fast lookups
CREATE INDEX IF NOT EXISTS idx_snapshots_event_mkt_ts
  ON odds_snapshots (match_id, market, ts_snapshot DESC);

-- C) optional: consensus blend
ALTER TABLE odds_consensus
  ADD COLUMN IF NOT EXISTS source_mix JSONB;

-- D) ensure xwalk exists (agent created it already; keep for completeness)
CREATE TABLE IF NOT EXISTS bookmaker_xwalk (
  canonical_name TEXT PRIMARY KEY,
  theodds_book_id TEXT,
  api_football_book_id TEXT,
  desk_group TEXT
);

Book ID rule going forward: for new API-Football rows, store book_id = 'apif:' || api_football_book_id. (Avoids collision with legacy numeric book_id from The Odds API. No PK migration needed.)

⸻

2) API-Football client (RapidAPI)
	•	Endpoints:
	•	/v3/odds?fixture={fixture_id} (pre-match)
	•	/v3/odds/live?fixture={fixture_id} (optional later)
	•	/v3/odds/bookmakers (seed xwalk)
	•	Headers: x-rapidapi-key, x-rapidapi-host

Agent tasks
	1.	Build ApiFootballClient with retries/backoff for 429/503 (exponential 1s, 2s, 4s, 8s).
	2.	Method: bookmakers() → returns [{"id":8,"name":"Bet365"}, ...]
	3.	Method: odds_by_fixture(fixture_id:int, live=False) → returns normalized response list.

⸻

3) Seed bookmaker crosswalk
	•	On boot + nightly: pull /v3/odds/bookmakers, upsert into bookmaker_xwalk:
	•	canonical_name = lower(trim(name))
	•	api_football_book_id = str(id)
	•	desk_group = canonicalize(name) (e.g., “bet365” → “bet365”, “william hill” → “williamhill”)
	•	(Optional) For legacy The Odds API rows, don’t mutate existing book_id now. We’ll dedupe by desk_group in consensus.

⸻

4) Mapping helpers
	•	Market: "Match Winner"/"1X2"/"Winner" → h2h; "Over/Under" → totals; "Asian Handicap"/"Handicap" → spread
	•	Selection: "Home"→H, "Draw"→D, "Away"→A. (Totals/spread parsing can be added later; H2H first.)

⸻

5) Ingestion (fixture-level)

ingest_fixture_odds_from_apifootball(fixture_id, match_id, league_id, kickoff_ts)
	•	For each bookmaker block:
	•	vendor_book_id = str(bookmaker.id)
	•	book_id = 'apif:' + vendor_book_id
	•	Map market & selection; parse odd → float; last_update → ts_snapshot
	•	Compute implied_prob = 1/odds_decimal
	•	secs_to_kickoff = kickoff_ts - ts_snapshot
	•	UPSERT into odds_snapshots with:
	•	match_id, league_id, book_id, market, outcome, ts_snapshot, secs_to_kickoff, odds_decimal, implied_prob, source='api_football', vendor_fixture_id, vendor_book_id
	•	After insertions per match:
	•	Optional: compute market_margin for H/D/A set and fill
	•	refresh_consensus_for_match(match_id)

Idempotency: upsert on the existing PK (match_id, book_id, market, outcome) + keep the newest ts_snapshot.

⸻

6) Gap-fill worker (post The Odds API sweep)

gap_fill_soccer_with_apifootball()
	•	Query: fixtures in next 72h (or historical batch) where exists_h2h=false or n_books < MIN_BOOKS in odds_snapshots.
	•	For each, call ingest_fixture_odds_from_apifootball(...).
	•	Run cadence: after your The Odds API job + again at T-60m to KO.
	•	Metrics to log: filled_fixtures_count, rows_inserted, rate_limit_hits, errors.

⸻

7) Consensus (no algorithm change)
	•	Include API-Football rows when building de-vigged trimmed mean.
	•	Desk dedupe: collapse books by desk_group via bookmaker_xwalk (e.g., Bet365 mobile/web same desk).
	•	Write source_mix JSONB, e.g., {"theodds":0.62,"api_football":0.38} by row share used in the calc.
	•	Update n_books to count distinct desk_group (not raw rows).

⸻

8) Backfill (historical) — controlled batches
	•	Target set: the 9,586 matches with zero authentic odds.
	•	Hit-rate assumption: start with 20–30%.
	•	Batcher: process in chunks of APIFOOTBALL.BACKFILL_BATCH (e.g., 200) with a 200–300ms pause per fixture to stay well under limits.
	•	Resume-safe: write progress to backfill_state table (match_id, status, attempts, last_error).
	•	After each batch, call rebuild_consensus_for(match_ids).

⸻

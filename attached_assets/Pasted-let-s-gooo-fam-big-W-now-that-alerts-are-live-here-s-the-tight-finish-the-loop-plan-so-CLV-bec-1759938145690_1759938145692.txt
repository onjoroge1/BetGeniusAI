# 4) Alert hygiene (what bit us; how we prevent repeats)

* **TZ normalization**: at DB-read boundary

  * Always coerce to aware UTC: `ts.replace(tzinfo=timezone.utc)` or use `datetime.fromisoformat(...).astimezone(timezone.utc)`.
  * Pick one field name everywhere: `ts_snapshot` (not `created_at`) + `odds_decimal` (not `odds_dec`).
* **Outcome codes**: standardize `'H'/'D'/'A'` in code and SQL; add a small enum helper.
* **Joins**: producer uses **fixtures** for KO/time; historical pieces can still read `matches` if needed—don’t cross the streams for “upcoming”.
